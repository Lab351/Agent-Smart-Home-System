# 编码智能体提示词 - 智能家居多 Agent 协作系统

## 项目总览

### 项目定位
构建一个**空间感知、去中心化、多 Agent 协作**的智能家居系统，通过 BLE Beacon、MQTT 和 MCP 协议实现：
- **自适应空间绑定**：用户携带 Personal Agent 进入房间，系统自动识别并提供该空间的控制能力
- **三层 Agent 协作**：Personal Agent（随身） + Room Agent（房间） + Central Agent（中央）
- **隐私优先**：个人数据本地处理，Room Agent 断网仍可自治运行

## 编码规范

### 1. 消息格式规范

#### 通用消息头
```json
{
  "message_id": "uuid-v4",
  "timestamp": "2024-01-15T10:30:00Z",
  "source_agent": "agent-id",
  "version": "1.0.0"
}
```

#### 控制消息
```json
{
  "message_id": "uuid-v4",
  "timestamp": "2024-01-15T10:30:00Z",
  "source_agent": "personal-agent-user1",
  "target_device": "light_1",
  "action": "on",
  "parameters": {
    "brightness": 80,
    "color_temp": 4000
  },
  "correlation_id": "optional-correlation-id"
}
```

#### 状态消息
```json
{
  "message_id": "uuid-v4",
  "timestamp": "2024-01-15T10:30:01Z",
  "agent_id": "room-agent-1",
  "room_state": {
    "mode": "idle",
    "occupancy": true,
    "environment": {...},
    "devices": {...}
  },
  "agent_status": "operational"
}
```

### 2. MQTT Topic 命名空间

```
room/{room_id}/
├── agent/{agent_id}/
│   ├── control/          # 控制命令
│   ├── state/            # 状态发布
│   ├── describe/         # 能力查询
│   ├── description/      # 能力响应
│   └── heartbeat/        # 心跳
├── robot/{robot_id}/
│   ├── control/
│   ├── state/
│   └── telemetry/
└── system/
    ├── discovery/
    └── error/

home/
├── state/                # 全局状态
├── policy/               # 策略更新
├── arbitration/          # 仲裁请求/响应
├── events/               # 系统事件
└── heartbeat/            # Central Agent 心跳
```

### 3. 策略规则格式

```yaml
policies:
  sleep_mode:
    light_max: low
    noise_max: minimum
    interruptible: false
    priority: 80
    triggers:
      - time: "22:00-07:00"
      - manual: true

  away_mode:
    all_devices: off
    security: armed
    priority: 90

  child_protection:
    content_filter: enabled
    volume_max: 50
    priority: 70
```

## 部署指南

### 硬件要求
- **Room Agent**：Jetson Nano/Orin, 树莓派 4/5, x86 Linux
- **Central Agent**：NAS/Mini PC/Jetson, 云端 VM（可选）
- **Beacon**：ESP32-C3/ESP32-S3（BLE 5.0）

### 网络配置
- **MQTT Broker**：局域网部署，仅监听 LAN 接口
- **TLS 加密**：生产环境强制 TLS 1.3
- **Topic ACL**：基于角色的访问控制

### 部署步骤
1. 启动 MQTT Broker
2. 配置 Room Agent（`config/room_agent.yaml`）
3. 配置 Central Agent（`config/default_config.yaml`）
4. 启动 Room Agent
5. 启动 Central Agent
6. 配置 qwen-backend Beacon API
7. 注册 Beacon

## 故障排查

### 常见问题

1. **Beacon 无法检测**
   - 检查 RSSI 阈值配置
   - 检查 Beacon 广播间隔
   - 检查 BLE 权限

2. **MQTT 连接失败**
   - 检查 Broker 地址与端口
   - 检查防火墙设置
   - 检查用户名/密码

3. **状态不同步**
   - 检查 Topic 订阅
   - 检查 QoS 配置
   - 检查网络延迟

## 关键设计原则

### Personal Agent
- **以人为中心**：所有行为由用户意图触发
- **低侵入性**：运行于随身设备，无需固定基础设施
- **隐私优先**：个人数据本地处理

### Room Agent
- **Room-first**：以物理空间为中心，而非设备或技能
- **State > Skill**：维护房间状态，而非提供大量接口
- **最小接口**：只暴露跨 Agent 协作的必要能力

### Central Agent
- **逻辑中心化，执行去中心化**：维护全局一致性，但不破坏房间自治
- **软约束优先**：通过规则引导，而非强制控制
- **默认不介入**：仅在必要时（冲突、违规）才介入
